影像分類模型訓練流程（Cavus 2CH / 4CH）

🔹 1. 資料蒐集
從網路收集已完成分類的影像資料（2CH / 4CH）
作為初始訓練資料集（pre-training dataset）

🔹 2. 模型選擇：DenseNet-121
📌 選擇原因

(1) 特徵重用（Feature Reuse）

每一層都接收所有前面層的輸出
低階特徵（邊緣、紋理）可持續利用
減少重複學習，提高效率

(2) 改善梯度消失

層與層之間有密集連接
梯度可直接回傳至前層
深層網路訓練更穩定

(3) 計算資源需求較低

相較部分深層模型參數較少
在資源有限（如 CPU）環境仍可運行（⚠️但訓練仍建議使用 GPU）


(4) 小樣本學習穩定（重要）

DenseNet 透過特徵重用機制，使模型在訓練資料較少時仍能維持穩定表現
減少過度依賴大量資料的需求
在初期僅有「網路公開資料」的情境下，仍可達到合理準確率

👉 適合本案例：

初期無亞東醫院資料 → 仍可建立可用 baseline 模型

🔹 3. 初始模型訓練與問題發現
將模型部署於亞東醫院實際資料
準確率僅約 70%
📌 問題原因
訓練資料與實際資料分佈不同（Domain Shift）
缺乏亞東醫院影像資料 → 泛化能力不足
🔹 4. 半自動標註流程（Human-in-the-loop）
使用初始模型預測大量未標註影像（萬筆）
提供給標註員進行修正與確認
收集高品質標註資料

👉 形成：
模型輔助標註 → 人工修正 → 回訓模型

🔹 5. 資料清洗與正規化
📌 問題觀察
影像中包含設備廠牌文字、UI資訊等雜訊
模型可能學到「錯誤特徵」（spurious correlation）
📌 解決方法
移除非必要資訊（文字、邊框、UI 設備廠商logo）
僅保留與 2CH / 4CH 有關影像的實際特徵
進行影像正規化（Normalization / Cropping）
🔹 6. 持續訓練與資料優化
反覆進行：
模型訓練
錯誤樣本分析
移除低品質或錯誤資料

👉 提升資料品質與模型穩定性

🔹 7. 模型泛化與成果
使用優化後資料重新訓練模型
在 10 萬筆影像測試
📈 成果：
準確率提升至 約 90%
大幅降低人工標註成本
提升整體標註效率
✅ 核心流程總結

👉 整體方法為：

預訓練模型 → 實際部署 → 發現問題 → 人機協作標註 → 資料清洗 → 持續迭代 → 泛化提升
